# Sprawl Community Plugins

Welcome to the Sprawl plugin ecosystem! Sprawl is designed to be highly extensible via WebAssembly (WASM) components. By writing a WASM plugin, you can add custom stack detection and safety rules to the Sprawl pipeline that will run identically on any platform.

## Architecture

Sprawl plugins are built on the **WASM Component Model**. 

We currently support extending the `Archaeologist` via the `StackDetector` interface. This allows Sprawl to accurately detect the languages, frameworks, and tools used in a given project, as well as providing safety heuristics before destructive actions (like nuking caches) occur.

## Getting Started

### The `stack-detector.wit` Interface
Every plugin must implement our provided WIT (WebAssembly Interface Type) definition. 

```wit
package sprawl:stack-detector;

interface detector {
    record stack-info {
        name: string,
        confidence: float32,
        dependencies: list<dependency>,
        reproducibility: reproducibility-verdict,
    }
    
    // ... check stack-detector.wit for the full interface ...
}
```

### Developing in Rust
We recommend writing Sprawl plugins in Rust using `cargo-component`:

1. Install `cargo-component`:
   ```bash
   cargo install cargo-component
   ```
2. Create a new component:
   ```bash
   cargo component new my-detector --lib
   ```
3. Copy our `stack-detector.wit` to a `wit/` folder in your component.
4. Implement the `Guest` trait generated by `wit-bindgen`.
5. Build your plugin:
   ```bash
   cargo component build --release
   ```

### Verifying and Installing

When you distribute your plugin, users can install it securely using:
```bash
sprawl plugin install path/to/my-detector.wasm --checksum <SHA-256-HASH>
```

Sprawl will verify the SHA-256 checksum to ensure the file hasn't been tampered with.

## Publishing Your Plugin

Presently, there is no centralized registry for Sprawl plugins. We encourage authors to:
- Publish your `.wasm` files to GitHub Releases.
- Provide the SHA-256 checksum prominently in your release notes.
- Tag your repository with `#sprawl-plugin` so the community can find it!
